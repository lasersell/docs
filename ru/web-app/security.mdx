Веб приложение использует модель безопасности, отличную от десктопного приложения. Понимание различий поможет вам принять обоснованное решение о том, какой клиент использовать.

## Модель доверия

| | Десктопное приложение | Веб приложение |
|--|------------|---------|
| **Модель** | Некастодиальная (локальная подпись) | Некастодиальная (делегированная подпись через Privy TEE) |
| **Кто подписывает** | Вы, локально | TEE Privy, от вашего имени |
| **Кто отправляет** | Вы выбираете (RPC, Helius и т.д.) | Серверы LaserSell через собственный RPC |
| **Расположение ключа** | Ваше устройство (зашифрован) | Trusted Execution Environment Privy |
| **LaserSell видит ваш ключ?** | Никогда | Никогда (TEE аппаратно изолирован) |

## Privy TEE

[Privy](https://privy.io) это компания, специализирующаяся на инфраструктуре кошельков и [приобретённая компанией Stripe](https://privy.io/blog/announcing-our-acquisition-by-stripe). Privy хранит ваш приватный ключ в **Trusted Execution Environment** (TEE), аппаратно изолированном анклаве. Ключ никогда не покидает TEE, даже при подписании транзакций. LaserSell запрашивает у Privy подпись транзакции, Privy подписывает её внутри TEE и возвращает только подписанный результат.

Это означает:
- Серверы LaserSell никогда не имеют доступа к вашему приватному ключу
- Компрометация сервера LaserSell не может привести к утечке вашего ключа
- Вы можете экспортировать свой ключ в любое время через защищённый интерфейс Privy

## Верификация владения кошельком

При регистрации кошелька веб приложение подписывает подтверждающее сообщение (`lasersell-register:<pubkey>:<timestamp>`) с помощью API `signMessage` от Privy. Эта подпись ed25519 проверяется серверами LaserSell, доказывая, что вы контролируете кошелёк. Это тот же процесс верификации, что и в десктопном приложении.

## Процесс транзакции

Когда срабатывает автоматическая продажа:

1. Сервер мониторинга LaserSell обнаруживает достижение порога стратегии
2. Создаётся неподписанная транзакция (как и в десктопном приложении)
3. LaserSell запрашивает у Privy **подпись** транзакции (только подпись, без отправки)
4. Privy подписывает внутри TEE и возвращает подписанную транзакцию
5. LaserSell отправляет подписанную транзакцию в Solana через собственный RPC
6. Вы получаете уведомление с подписью транзакции

Ни на одном этапе серверы LaserSell не видят ваш приватный ключ. Подпись и хранение ключей полностью осуществляются в инфраструктуре Privy.

## Защита данных

| Данные | Хранение | Доступ |
|------|---------|--------|
| API ключи | Зашифрованы в Supabase | Безопасность на уровне столбцов предотвращает доступ браузера к необработанным ключам |
| Приватные ключи кошельков | Privy TEE | Никогда не доступны для LaserSell |
| Состояние торговой сессии | Supabase (защищено RLS) | Видны только ваши собственные сессии |
| Уведомления | Supabase (защищено RLS) | Видны только ваши собственные уведомления |
| Push подписки | Supabase (защищено RLS) | Управляются пользователем |

## Удаление вашего ключа из Privy

Если вы хотите полностью удалить свой приватный ключ из инфраструктуры Privy:

### Импортированные кошельки
1. Перейдите на страницу **Wallet** в панели управления
2. Нажмите значок корзины на карточке импортированного кошелька
3. Это удаляет кошелёк из LaserSell И удаляет ключ из TEE Privy
4. Ключ безвозвратно уничтожается в анклаве

### Встроенные кошельки
Встроенные кошельки привязаны к вашей учётной записи Privy. Чтобы удалить их:
1. **Сначала экспортируйте приватный ключ** (нажмите «Export Key» на карточке кошелька)
2. Сохраните экспортированный ключ в собственном кошельке (Phantom, Backpack и т.д.)
3. Обратитесь в [поддержку](https://docs.lasersell.io/getting-support/support), чтобы запросить удаление аккаунта Privy
4. После удаления аккаунта Privy ключ встроенного кошелька безвозвратно уничтожается

<Warning>
  Удаление кошелька или аккаунта Privy необратимо. Если вы не экспортировали приватный ключ, вы навсегда потеряете доступ к средствам в этом кошельке.
</Warning>

## Рекомендации

- Используйте **выделенный торговый кошелёк** для веб торговли вместо основного кошелька для хранения средств
- **Отключайте веб торговлю** при переключении на десктопное приложение, чтобы избежать дублирования попыток продажи
- **Экспортируйте приватный ключ** после импорта кошелька в качестве резервной копии
- Проверяйте настройку **максимального проскальзывания** в стратегии выхода (по умолчанию 5%) для защиты от MEV при автоматическом исполнении
