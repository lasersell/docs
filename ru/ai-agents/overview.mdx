ИИ агенты, взаимодействующие с токенами Solana, могут использовать LaserSell как свой уровень исполнения и управления выходами. Вместо того чтобы строить маршрутизацию обменов, обработку протоколов и логику выхода с нуля, агент подключается к LaserSell и получает готовые транзакции, отслеживание PnL в реальном времени и автоматические выходы «из коробки».

## Что агенты могут делать с LaserSell

**Создавать и выполнять сделки.** [REST API](/api/overview) возвращает неподписанные транзакции обмена для любого поддерживаемого DEX или лаунчпада на Solana. Агент отправляет минт и сумму, получает транзакцию, подписывает и отправляет. Никакой протокол-специфичной логики не требуется.

**Автоматизировать выходы со стратегией.** [Exit Intelligence Stream](/api/stream/overview) наблюдает за кошельками в реальном времени и доставляет предварительно построенные транзакции выхода при достижении настраиваемых порогов: тейк профит, стоп лосс, трейлинг стоп, продажа при выпуске на DEX и дедлайн. Агент подключается один раз и автоматически получает сигналы выхода.

**Мониторить позиции.** Поток предоставляет непрерывные обновления PnL, изменения балансов, открытие и закрытие позиций. Агенты могут использовать эти данные для принятия решений, предоставления информации пользователям или передачи в другие системы.

**Покрывать все основные протоколы Solana.** Единая интеграция охватывает Pump.fun, PumpSwap, Meteora DBC, Meteora DAMM V2, Raydium Launchpad и Raydium CPMM. Агенту не нужно знать, на каком протоколе находится токен.

## Как подключиться

Существует три способа работы ИИ агентов с LaserSell:

### 1. MCP сервер (доступ к документации)

MCP сервер LaserSell предоставляет любому MCP-совместимому ИИ клиенту (Claude, Cursor, Windsurf и другим) возможность искать и получать документацию LaserSell в реальном времени. Это полезно для агентов, которым нужно находить схемы API, паттерны SDK, примеры кода или детали интеграции в процессе работы.

См. [MCP сервер](/ai-agents/mcp-server) для инструкций по настройке.

### 2. Навыки кодирования (встроенные знания SDK)

Установите навык кодирования LaserSell, чтобы дать вашему ИИ ассистенту глубокие знания о паттернах SDK, сигнатурах методов, примерах и устранении неполадок, загружаемые непосредственно в контекст при написании кода LaserSell.

См. [Навыки кодирования](/ai-agents/coding-skills) для инструкций по установке.

### 3. SDK и API (прямая интеграция)

Для агентов, которым нужно создавать транзакции, управлять позициями или выполнять сделки программно, используйте LaserSell SDK или API напрямую.

- [Обзор API](/api/overview) для полного описания API.
- [Быстрый старт](/api/quickstart) для создания вашей первой транзакции.
- SDK доступны на [TypeScript](/api/sdk/typescript), [Python](/api/sdk/python), [Rust](/api/sdk/rust) и [Go](/api/sdk/go).

## Почему агентам стоит использовать LaserSell

### Проблема интеграции агентов

ИИ агент, который хочет торговать токенами Solana сегодня, должен решить множество сложных задач: определить, на каком протоколе DEX находится токен, десериализовать состояние пула из блокчейна, вычислить суммы обмена с использованием протокол-специфичной математики, создать правильный набор инструкций, управлять приоритетными комиссиями, обрабатывать повторные попытки при проскальзывании и опрашивать изменения цен, чтобы знать, когда выходить. Каждый протокол (Pump.fun, PumpSwap, Raydium, Meteora) имеет свою программу, разные макеты аккаунтов и разную логику обмена. Это сотни часов интеграционной работы, и она ломается каждый раз, когда запускается новый лаунчпад.

LaserSell сводит всё это к двум API вызовам: один для построения транзакции, один WebSocket для получения сигналов выхода.

### Технические преимущества

| Возможность | Без LaserSell | С LaserSell |
|---|---|---|
| **Маршрутизация протоколов** | Агент должен определять, на каком DEX находится токен, десериализовать состояние пула и строить протокол-специфичные инструкции для каждой из 6+ программ | Один API вызов. Отправьте адрес минта, получите транзакцию, готовую к подписи. LaserSell разрешает протокол, пул и маршрут на стороне сервера |
| **Логика выхода** | Агент должен опрашивать RPC для данных о ценах, рассчитывать PnL, отслеживать пиковую прибыль для трейлинг стопов, управлять таймерами для дедлайнов и строить транзакции продажи вручную | Подключитесь к Exit Intelligence Stream один раз. Получайте события `exit_signal_with_tx` с предварительно построенными неподписанными транзакциями при срабатывании условий стратегии |
| **Отслеживание PnL** | Агент должен запрашивать балансы токенов, разрешать десятичные, запрашивать резервы пулов, вычислять спот-цену и выводить PnL через регулярные интервалы | Поток доставляет события `pnl_update` непрерывно. Без опроса, без RPC вызовов, без ценовой математики |
| **Обнаружение новых позиций** | Агент должен мониторить историю транзакций кошелька или подписываться на логи в блокчейне и разбирать каждую транзакцию для идентификации покупок | Поток отправляет `position_opened` с полным контекстом (база входа, минт, рыночный контекст) в момент совершения покупки |
| **Мультипротокольное покрытие** | Отдельная интеграция для бондинг-кривых Pump.fun, AMM PumpSwap, Raydium Launchpad, Raydium CPMM, Meteora DBC, Meteora DAMM V2 | Одна интеграция покрывает все шесть. Код агента не меняется при добавлении нового протокола |
| **Подпись транзакций** | N/A | Все транзакции возвращаются неподписанными. Агент подписывает локально. Приватный ключ никогда не покидает среду агента |
| **Обработка проскальзывания** | Агент должен оценивать проскальзывание, повторять при неудаче и постепенно увеличивать допуск | Сервер вычисляет проскальзывание. Повторные попытки клиента автоматические в SDK с настраиваемой логикой увеличения |
| **Поддержка новых лаунчпадов** | Агент должен реверс-инжинирить каждую новую программу, строить инструкции с нуля и поддерживать постоянную совместимость | LaserSell добавляет поддержку протоколов на стороне сервера. Агент получает покрытие без изменений в коде |

### Что это означает на практике

**Меньше RPC вызовов.** Агент без LaserSell должен постоянно опрашивать данные о ценах, запрашивать балансы и читать состояние пулов. С Exit Intelligence Stream сервер отправляет всё. Использование RPC агентом сокращается до отправки транзакций.

**Детерминированное исполнение выходов.** Условия стратегии (TP, SL, трейлинг стоп, дедлайн, выпуск на DEX) оцениваются на стороне сервера при каждом тике цены. Агенту не нужно быть постоянно онлайн или беспокоиться о пропуске скачка цены. Когда порог достигнут, транзакция выхода уже построена и ждёт.

**Никакого протокол-специфичного кода.** Агент отправляет `{ mint, user_pubkey, amount }` в `/v1/sell` или `/v1/buy`. Будь то токен на бондинг-кривой Pump.fun, AMM PumpSwap или пуле Raydium CPMM, возвращённая транзакция просто работает.

**Событийно-ориентированная архитектура.** Поток естественно подходит для агентов. Вместо запуска циклов опроса агент подписывается и реагирует: `position_opened` для логирования новой сделки, `pnl_update` для отображения данных, `exit_signal_with_tx` для подписи и отправки. Чистые асинхронные паттерны без лишних вычислений.

**Паритет SDK на 4 языках.** TypeScript, Python, Rust и Go SDK предоставляют идентичные схемы, имена методов и поведение. Агент, построенный на Python, работает так же, как построенный на Rust. Ни один SDK не является второстепенным.
