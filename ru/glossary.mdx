title: "Глоссарий"
description: "Определения ключевых терминов, аббревиатур и концепций, используемых в документации LaserSell."
---

## A

**AMM (Automated Market Maker)**
Протокол децентрализованной биржи, использующий математические формулы для определения цен токенов на основе резервов пула, обеспечивая обмен токенов без книг заказов. PumpSwap, Meteora DAMM V2 и Raydium CPMM являются примерами AMM, поддерживаемых LaserSell.

**Argon2id**
Алгоритм хеширования паролей с высокими требованиями к памяти, используемый LaserSell для вывода ключей шифрования из вашей парольной фразы. Делает атаки перебором на ваше хранилище ключей вычислительно дорогими.

**Associated Token Account (ATA)**
Производный аккаунт в блокчейне, который содержит определённый SPL токен для данного адреса кошелька. Создаётся автоматически, когда кошелёк впервые получает токен.

**Astralane Iris**
Шлюз отправки транзакций, поддерживающий региональные эндпоинты для оптимизации задержки. Один из доступных [целей отправки](/api/transactions/send-targets) в SDK LaserSell. Требуется API ключ Astralane.

**Атомарные единицы**
Наименьшая неделимая единица токена. Для SOL атомарная единица — лампорт (1 SOL = 1 000 000 000 лампортов). Другие токены определяют собственную десятичную точность.

## B

**Balance Update**
Серверное событие (`balance_update`), отправляемое Exit Intelligence Stream при изменении баланса SOL или токенов вашего кошелька. См. [Серверные события](/api/stream/server-events).

**Базисные пункты (BPS)**
Единица, равная 1/100 процента. 100 bps = 1%. Используется в LaserSell API для настройки допуска проскальзывания и партнёрских комиссий.

**Бондинг-кривая**
Математическая формула, используемая лаунчпадами вроде Pump.fun для определения цены токена на основе его обращающегося предложения. Токены на бондинг-кривой ещё не мигрировали в пул AMM.

## C

**Холодное хранилище**
Офлайн-хранение кошелька (например, аппаратный кошелёк), используемое для долгосрочного хранения. Регулярно выводите прибыль из горячего кошелька в холодное хранилище.

**Configure Message**
Первое клиентское сообщение, отправляемое после подключения к Exit Intelligence Stream. Указывает адреса ваших кошельков и параметры стратегии. Сервер отвечает событием `configured`. См. [Клиентские сообщения](/api/stream/client-messages).

## D

**Дедлайн (Deadline Timeout)**
Временной лимит (в секундах), который принудительно закрывает позицию, если никакое другое условие стратегии (тейк профит, стоп лосс, трейлинг стоп) не сработало в течение настроенного времени. Предотвращает блокировку капитала в стагнирующих позициях.

## E

**Exit Intelligence**
Основная технология LaserSell. Система реального времени, которая отслеживает позиции кошелька, оценивает условия стратегии при каждом обновлении цены и доставляет предварительно построенные неподписанные транзакции выхода при достижении пороговых значений.

**Exit Intelligence Stream**
Постоянное WebSocket-соединение с `wss://stream.lasersell.io/v1/ws`, обеспечивающее отслеживание позиций в реальном времени, обновления PnL и автоматические сигналы выхода. См. [Обзор Stream](/api/stream/overview).

**Exit Signal**
Серверное событие (`exit_signal_with_tx`), содержащее предварительно построенную неподписанную транзакцию продажи, срабатывающее при выполнении условия стратегии. Самое важное событие в Exit Intelligence Stream. См. [Серверные события](/api/stream/server-events).

**ExitApiClient**
Клиент SDK для выполнения REST API вызовов для построения транзакций покупки и продажи по запросу через `/v1/buy` и `/v1/sell`.

## G

**Выпуск на DEX (Graduation)**
Момент, когда токен мигрирует с бондинг-кривой (например, Pump.fun) в пул AMM (например, PumpSwap). Ценовая динамика часто значительно меняется при выпуске на DEX.

## H

**`hello_ok`**
Первое серверное сообщение, получаемое после подключения к Exit Intelligence Stream. Содержит идентификатор сессии и лимиты тарифа. См. [Жизненный цикл соединения](/api/stream/connection-lifecycle).

**Helius Sender**
Сервис отправки транзакций, маршрутизирующий через эндпоинт `/fast` Helius для оптимизированного попадания транзакций в Solana. API ключ не требуется. [Цель отправки](/api/transactions/send-targets) по умолчанию для LaserSell.

**Горячий кошелёк**
Любой кошелёк, подключённый к интернету для активной торговли. LaserSell создаёт локальный горячий кошелёк на вашем устройстве для быстрого исполнения сделок.

## J

**Jito Tip**
Прямая оплата валидаторам Solana для приоритизации вашей транзакции над другими в том же блоке. Это сетевой расход, а не комиссия LaserSell.

## K

**Хранилище ключей (Keystore)**
Формат зашифрованного файла, используемый клиентом LaserSell для локального хранения ваших приватных ключей. Использует шифрование XChaCha20-Poly1305 с ключом, выведенным через Argon2id. См. [Шифрование кошелька](/security/wallet-encryption).

## L

**Лампорт**
Наименьшая единица SOL. 1 SOL = 1 000 000 000 лампортов. Поля API, такие как `amount_in_total`, `tip_lamports` и `partner_fee_lamports`, указываются в лампортах.

**Lane Splitting (Разделение каналов)**
Функция (только в TypeScript SDK), которая разделяет Exit Intelligence Stream на два приёмника: высокоприоритетный канал для сигналов выхода и событий позиций и низкоприоритетный канал для обновлений PnL. Предотвращает блокировку времязависимых сигналов выхода частыми обновлениями PnL.

## M

**Market Context**
Метаданные, описывающие пул DEX и протокол, используемый для рынка токена, включая адреса пулов и конфигурацию. Возвращается в событиях потока и необязательно передаётся в запросы построения для пропуска серверного разрешения. См. [Market Context](/api/reference/market-context).

**Мемкоин**
Криптовалюта, часто вдохновлённая интернет-культурой, характеризующаяся высокой волатильностью и спекулятивной торговлей. Основной сегмент рынка, для которого создан LaserSell.

**Meteora DBC (Dynamic Bonding Curve)**
Протокол бондинг-кривой Meteora для токенов в начальной фазе запуска. Поддерживается LaserSell.

**Meteora DAMM V2 (Dynamic AMM V2)**
Динамический автоматизированный маркетмейкер Meteora для токенов, мигрировавших с бондинг-кривой. Поддерживается LaserSell.

**Mode (Маршрутизация)**
Поле в запросах `/v1/buy` и `/v1/sell` со значениями `"fast"` или `"secure"`. Режим fast оптимизирует скорость; режим secure добавляет дополнительные проверки подтверждения. Не путать с Send Mode.

## N

**Некастодиальный**
Архитектура, при которой пользователи сохраняют полный контроль над своими приватными ключами и средствами. LaserSell никогда не хранит ключи на своих серверах. Все транзакции возвращаются неподписанными и подписываются локально пользователем или его приложением.

## P

**Партнёрская комиссия**
Необязательная комиссия, которую интеграторы могут добавить к любой транзакции покупки или продажи, направляемая атомарно в указанный кошелёк как часть обмена в блокчейне. Настраивается в базисных пунктах или фиксированных лампортах для каждого запроса. Доступна на тарифах Advanced и Enterprise. См. [Партнёрские комиссии](/api/reference/partner-fees).

**Парольная фраза**
Пользовательский пароль, необходимый для разблокировки зашифрованного хранилища ключей. Если эта парольная фраза утеряна, кошелёк не может быть расшифрован.

**PnL (Profit and Loss)**
Текущая расчётная прибыль или убыток по позиции. Exit Intelligence Stream доставляет события `pnl_update` с регулярными интервалами с обновлёнными значениями.

**`pnl_update`**
Серверное событие, содержащее последний расчёт PnL для отслеживаемой позиции. Доставляется с регулярными интервалами. Высокая частота доставки является причиной существования разделения каналов. См. [Серверные события](/api/stream/server-events).

**Position Handle**
Ссылочный объект, возвращаемый событиями `StreamSession`, который идентифицирует отслеживаемую позицию по её ID, токен-аккаунту, минту и кошельку. Используется для закрытия позиций и запроса сигналов выхода.

**`position_opened`**
Серверное событие, срабатывающее когда Exit Intelligence Stream обнаруживает новую покупку токена в отслеживаемом кошельке. Содержит ID позиции, минт, базу входа и рыночный контекст. См. [Серверные события](/api/stream/server-events).

**`position_closed`**
Серверное событие, срабатывающее когда отслеживаемая позиция полностью закрыта или удалена. См. [Серверные события](/api/stream/server-events).

**Priority Fee**
Стандартная комиссия блокчейна Solana, уплачиваемая валидаторам за вычислительный приоритет. Отдельно от Jito tips.

**Приватный ключ**
Криптографический ключ, дающий контроль над активами кошелька. Клиент LaserSell шифрует его локально и никогда не передаёт. API и SDK никогда не требуют приватный ключ; все транзакции возвращаются неподписанными.

**Pump.fun**
Лаунчпад токенов Solana, где токены начинают торговаться на бондинг-кривой до перехода на AMM.

**PumpSwap**
AMM, на который мигрируют токены Pump.fun после выхода с бондинг-кривой. Поддерживается LaserSell.

## R

**Лимиты запросов (Rate Limits)**
Ограничения на количество запросов или ресурсов, которые вы можете использовать, определяемые вашим тарифным планом. Применяются как к REST API (запросов в минуту), так и к Exit Intelligence Stream (кошельки, позиции, сессии). См. [Лимиты и тарифы](/api/reference/rate-limits).

**Raydium CPMM (Constant Product Market Maker)**
Стандартные пулы AMM Raydium. Поддерживается LaserSell.

**Raydium Launchpad**
Платформа запуска токенов Raydium. Поддерживается LaserSell.

**Комиссия маршрутизации (Routing Fee)**
Комиссия за сделку, применяемая к каждому ордеру, выполненному через LaserSell Router. Ставка зависит от вашего тарифного плана: 2% (бесплатный), 0.9% (Professional), 0.5% (Advanced) или индивидуальная (Enterprise). Применяется атомарно в блокчейне. См. [Структура комиссий](/pricing/fee-structure).

**RPC (Remote Procedure Call)**
Нода, служащая шлюзом к блокчейну Solana. LaserSell использует RPC соединение для чтения данных из сети и трансляции транзакций. Для продакшена рекомендуется частный RPC эндпоинт.

**Rug Pull**
Сценарий, при котором ликвидность токена выводится или продажа отключается, вызывая обвал цены. Функции стоп лосса, трейлинг стопа и продажи при выпуске на DEX в LaserSell помогают ограничить потери.

## S

**Сид-фраза**
Мнемоническая фраза из 12 или 24 слов, используемая для генерации или восстановления кошелька. Отображается только один раз при первичном создании кошелька.

**Продажа при выпуске на DEX (Sell on Graduation)**
Условие стратегии, которое автоматически закрывает позицию в момент миграции токена с бондинг-кривой в пул AMM. Защищает от пост-миграционных дампов.

**Send Mode**
Поле в запросах построения (`"rpc"`, `"helius_sender"` или `"astralane"`), указывающее серверу, для какого контекста отправки оптимизировать транзакцию. Должно соответствовать выбранной цели отправки.

**Send Target**
Конфигурация, указывающая SDK, куда отправлять подписанную транзакцию. Варианты: Default RPC, Custom RPC, Helius Sender, Astralane Iris. См. [Цели отправки](/api/transactions/send-targets).

**Проскальзывание (Slippage)**
Разница между ожидаемой ценой сделки и фактической ценой исполнения. Высокая волатильность и низкая ликвидность увеличивают проскальзывание.

**Стоп лосс (Stop Loss)**
Условие стратегии, которое автоматически продаёт позицию, когда цена падает на настроенный процент от цены входа. Ограничивает потери по отдельной сделке.

**StreamClient**
Низкоуровневый клиент SDK для Exit Intelligence Stream. Управляет WebSocket-соединением, переподключением и кадрированием сообщений. Возвращает необработанные серверные сообщения. См. [Жизненный цикл соединения](/api/stream/connection-lifecycle).

**StreamSession**
Высокоуровневая обёртка SDK над `StreamClient`. Обеспечивает автоматическое отслеживание позиций, таймеры дедлайнов, типизированные события с хэндлами позиций и методы обновления стратегии. Рекомендуется для большинства интеграций.

## T

**Тейк профит (Target Profit)**
Условие стратегии, вызывающее автоматическую продажу, когда PnL позиции достигает настроенного процента прибыли.

**Трейлинг стоп (Trailing Stop)**
Динамический стоп лосс, который отслеживает наибольшую прибыль, наблюдаемую с момента открытия позиции. Если прибыль падает от этого пика на настроенный процент, срабатывает сигнал выхода. Фиксирует прибыль по позициям, которые выросли перед разворотом.

## U

**USD1**
Стейблкоин, поддерживаемый как альтернативный выходной актив для транзакций продажи (вместо SOL).

**Неподписанная транзакция (Unsigned Transaction)**
Закодированная в base64 транзакция Solana `VersionedTransaction`, возвращаемая LaserSell API. Должна быть подписана локально вашим приватным ключом перед отправкой. LaserSell никогда не выполняет подпись.

## V

**VersionedTransaction**
Формат транзакции Solana, поддерживающий версионные макеты инструкций и таблицы поиска адресов. Все транзакции, возвращаемые LaserSell API, используют этот формат.

## W

**WebSocket (WSS)**
Протокол связи, обеспечивающий двунаправленные потоки данных в реальном времени. Exit Intelligence Stream использует WSS для доставки событий позиций и сигналов выхода с минимальной задержкой.

## X

**XChaCha20-Poly1305**
Алгоритм аутентифицированного шифрования, используемый клиентом LaserSell для защиты приватных ключей в файле хранилища. Обеспечивает как конфиденциальность, так и проверку целостности.
