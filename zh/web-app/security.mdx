Web App 使用与桌面应用不同的安全模型。了解两者之间的差异有助于你做出明智的选择，决定使用哪个客户端。

## 信任模型

| | 桌面应用 | Web App |
|--|------------|---------|
| **模型** | 非托管（本地签名） | 非托管（通过 Privy TEE 委托签名） |
| **签名者** | 你，在本地完成 | Privy 的 TEE，代表你完成 |
| **提交者** | 你自行选择（RPC、Helius 等） | LaserSell 的服务器通过自有 RPC |
| **密钥位置** | 你的设备（加密存储） | Privy 的可信执行环境 |
| **LaserSell 能看到你的密钥吗？** | 永远不能 | 永远不能（TEE 是硬件隔离的） |

## Privy TEE

[Privy](https://privy.io) 是一家钱包基础设施公司，已被 [Stripe 收购](https://privy.io/blog/announcing-our-acquisition-by-stripe)。Privy 将你的私钥存储在**可信执行环境**（TEE）中，这是一个硬件隔离的安全区域。密钥永远不会离开 TEE，即使在签署交易时也是如此。LaserSell 请求 Privy 签署交易，Privy 在 TEE 内部完成签名，并只返回已签名的结果。

这意味着：
- LaserSell 服务器永远无法访问你的私钥
- LaserSell 服务器被入侵也不会泄露你的密钥
- 你可以随时通过 Privy 的安全界面导出你的密钥

## 钱包所有权验证

当你注册钱包时，Web App 会使用 Privy 的 `signMessage` API 签署一条证明消息（`lasersell-register:<pubkey>:<timestamp>`）。这个 ed25519 签名由 LaserSell 的服务器验证，以证明你拥有该钱包的控制权。这与桌面应用使用的验证流程相同。

## 交易流程

当自动卖出触发时：

1. LaserSell 的监控服务器检测到策略阈值被触发
2. 构建一笔未签名的交易（与桌面应用相同）
3. LaserSell 请求 Privy **签署**该交易（仅签名，不提交）
4. Privy 在 TEE 内部完成签名并返回已签名的交易
5. LaserSell 通过自有 RPC 将已签名的交易提交到 Solana
6. 你收到一条包含交易签名的通知

在整个过程中，LaserSell 的服务器都看不到你的私钥。签名和密钥存储完全在 Privy 的基础设施内完成。

## 数据保护

| 数据 | 存储位置 | 访问控制 |
|------|---------|--------|
| API 密钥 | Supabase 加密存储 | 列级安全防止浏览器访问原始密钥 |
| 钱包私钥 | Privy TEE | LaserSell 永远无法访问 |
| 交易会话状态 | Supabase（RLS 保护） | 仅你自己的会话可见 |
| 通知 | Supabase（RLS 保护） | 仅你自己的通知可见 |
| 推送订阅 | Supabase（RLS 保护） | 用户自行管理 |

## 从 Privy 中移除你的密钥

如果你想从 Privy 的基础设施中完全移除你的私钥：

### 已导入的钱包
1. 前往仪表板的**钱包**页面
2. 点击已导入钱包卡片上的垃圾桶图标
3. 这会将钱包从 LaserSell 中移除，并从 Privy 的 TEE 中删除密钥
4. 密钥在安全区域中被永久销毁

### 内嵌钱包
内嵌钱包与你的 Privy 账户绑定。要移除它们：
1. **先导出你的私钥**（点击钱包卡片上的「导出密钥」）
2. 将导出的密钥安全存储在你自己的钱包中（Phantom、Backpack 等）
3. 联系[支持团队](https://docs.lasersell.io/getting-support/support)请求删除 Privy 账户
4. 一旦 Privy 账户被删除，内嵌钱包的密钥将被永久销毁

<Warning>
  删除钱包或 Privy 账户是不可逆的。如果你没有导出私钥，你将永久失去对该钱包中资金的访问权限。
</Warning>

## 建议

- 使用**专门的交易钱包**进行 Web 交易，而不是你的主要持仓钱包
- 切换到桌面应用时**关闭 Web 交易**，以避免重复卖出操作
- 导入钱包后**导出你的私钥**作为备份
- 检查退出策略的**最大滑点**设置（默认 5%），以在无人值守执行时防范 MEV
