## 消息类型

服务器发送带有 `type` 判别器的消息。所有 SDK 将其解析为类型化对象。

| 类型                    | 优先级 | 说明                                    |
|-------------------------|----------|------------------------------------------------|
| `hello_ok`              | 高     | 连接已接受，会话已开始。          |
| `pong`                  | 高     | 对客户端 ping 的响应。                       |
| `error`                 | 高     | 服务器错误通知。                     |
| `balance_update`        | 高     | 被追踪钱包的代币余额变更。    |
| `position_opened`       | 高     | 检测到新仓位并开始追踪。    |
| `position_closed`       | 高     | 仓位追踪结束。                       |
| `exit_signal_with_tx`   | 高     | 退出阈值已满足；包含未签名交易。      |
| `pnl_update`            | 低      | 定期盈亏快照。             |
| `liquidity_snapshot`    | 低      | 滑点带和流动性趋势（Tier 1+）。  |
| `trade_tick`            | 低      | 被追踪仓位的实时交易通知。 |

关于每种消息类型的完整 JSON 模式、字段说明和使用示例，请参阅[英文版](/api/stream/server-events)。所有字段名、类型和 JSON 结构与英文版完全一致。

## 关键事件说明

**`hello_ok`** — 连接打开后立即发送一次。包含会话 ID 和等级限制。

**`balance_update`** — 被追踪钱包中代币余额变更时发送（新到达、转账、卖出）。

**`position_opened`** — 检测到新仓位并开始追踪。包含仓位 ID、代币地址、成本基础和市场上下文。

**`position_closed`** — 仓位已从追踪中移除。包含关闭原因。

**`exit_signal_with_tx`** — 最重要的事件。当仓位满足退出条件时发送。包含预构建的未签名交易，可直接签名。

**`pnl_update`** — 被追踪仓位的定期盈亏快照。按 `limits.pnl_flush_ms` 定义的间隔发送。

**`liquidity_snapshot`** — 仅 Tier 1+。实时滑点带和流动性趋势。帮助你了解在给定滑点阈值下可以卖出多少仓位。

**`trade_tick`** — 被追踪仓位代币上发生交易时的实时通知。
