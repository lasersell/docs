LaserSell 웹 앱은 자동 거래를 브라우저로 가져옵니다. 청산 전략을 설정하고, 지갑에 자금을 입금하면, LaserSell이 브라우저를 닫아도 연중무휴 24시간 자동으로 거래를 실행합니다.

## 작동 방식

<Steps>
  <Step title="지갑 설정">
    로그인하면 [Privy](https://privy.io)에 의해 보안 지갑이 자동으로 생성됩니다. 기존 지갑을 가져올 수도 있습니다. 개인키는 Trusted Execution Environment(TEE)에 저장되며 LaserSell 서버에는 전달되지 않습니다.
  </Step>
  <Step title="지갑에 자금 입금">
    외부 지갑(Phantom, Backpack 등)에서 지갑 주소로 SOL을 전송하세요. 지갑 주소와 잔액은 Wallet 페이지에서 확인할 수 있습니다.
  </Step>
  <Step title="전략 설정">
    청산 전략을 설정하세요: 목표 수익, 손절매, 트레일링 스탑, 최대 슬리피지. 이 규칙에 따라 LaserSell이 자동으로 포지션을 매도합니다.
  </Step>
  <Step title="거래 시작">
    "Start Web Trading"을 클릭하면 LaserSell이 지갑 모니터링을 시작합니다. 포지션이 전략 임계값에 도달하면 지갑을 사용하여 자동으로 거래가 실행됩니다.
  </Step>
</Steps>

## 웹 앱 vs 데스크톱 앱

두 클라이언트 모두 동일한 LaserSell 인프라에 연결되지만, 내부 작동 방식이 다릅니다.

| 기능 | 데스크톱 앱 | 웹 앱 |
|---------|------------|---------|
| **서명** | 로컬 키페어 | Privy TEE 지갑 (위임 서명) |
| **오프라인 실행** | 아니요 (앱이 열려 있어야 함) | 예 (부재 중에도 서버가 실행) |
| **지갑** | 로컬에서 가져오기/생성 | Privy 임베디드 또는 가져오기 |
| **카피 트레이딩** | 예 | 예 |
| **포지션 모니터링** | 예 | 예 |
| **코인 매수** | 예 | 아직 미지원 |
| **토큰 검색** | 예 | 아직 미지원 |

## 핵심 개념

### Privy 지갑

웹 트레이딩 지갑은 [Stripe에 인수된](https://privy.io/blog/announcing-our-acquisition-by-stripe) 지갑 인프라 회사인 [Privy](https://privy.io)가 관리합니다. Privy는 개인키를 Trusted Execution Environment(TEE), 즉 하드웨어로 격리된 보안 영역에 저장하여 LaserSell을 포함한 그 누구도 키에 직접 접근할 수 없도록 합니다.

거래를 실행해야 할 때, LaserSell이 Privy에 트랜잭션 서명을 요청합니다. Privy는 TEE 내부에서 서명하고 서명된 트랜잭션을 반환합니다. 그런 다음 LaserSell이 이를 Solana 네트워크에 제출합니다.

### 상시 모니터링

웹 트레이딩이 활성화되면, LaserSell은 사용자를 대신하여 모니터링 인프라와 지속적인 연결을 유지합니다. 포지션은 실시간으로 추적되며, 전략 임계값에 도달하면 즉시 청산 신호가 실행됩니다.

대시보드 헤더의 전원 아이콘을 사용하여 웹 트레이딩을 켜고 끌 수 있습니다. 데스크톱 앱으로 전환할 때는 동일한 지갑에 대한 중복 매도 시도를 방지하기 위해 웹 트레이딩을 끄세요.

### 세션 상태

| 상태 | 의미 |
|-------|--------------|
| **Active** | 스트림 연결됨, 포지션 모니터링 중, 거래 자동 실행 |
| **Paused** | 일시 중지됨 (오류 또는 수동 일시 중지) |
| **Stopped** | 웹 트레이딩 꺼짐, 리소스 소비 없음 |
