웹 앱은 데스크톱 앱과 다른 보안 모델을 사용합니다. 차이점을 이해하면 어떤 클라이언트를 사용할지 정보에 기반한 결정을 내릴 수 있습니다.

## 신뢰 모델

| | 데스크톱 앱 | 웹 앱 |
|--|------------|---------|
| **모델** | 비수탁형 (로컬 서명) | 비수탁형 (Privy TEE를 통한 위임 서명) |
| **서명 주체** | 사용자, 로컬에서 | Privy의 TEE, 사용자를 대신하여 |
| **제출 주체** | 사용자 선택 (RPC, Helius 등) | LaserSell 서버가 자체 RPC를 통해 |
| **키 위치** | 사용자 기기 (암호화됨) | Privy의 Trusted Execution Environment |
| **LaserSell이 키를 보나요?** | 절대 아님 | 절대 아님 (TEE는 하드웨어 격리됨) |

## Privy TEE

[Privy](https://privy.io)는 [Stripe에 인수된](https://privy.io/blog/announcing-our-acquisition-by-stripe) 지갑 인프라 회사입니다. Privy는 개인키를 **Trusted Execution Environment**(TEE), 즉 하드웨어로 격리된 보안 영역에 저장합니다. 키는 트랜잭션에 서명할 때에도 TEE를 벗어나지 않습니다. LaserSell이 Privy에 트랜잭션 서명을 요청하면, Privy가 TEE 내부에서 서명하고 서명된 결과만 반환합니다.

이것은 다음을 의미합니다:
- LaserSell 서버는 개인키에 접근할 수 없습니다
- LaserSell 서버가 침해되어도 키가 유출되지 않습니다
- Privy의 보안 UI를 통해 언제든지 키를 내보낼 수 있습니다

## 지갑 소유권 확인

지갑을 등록할 때, 웹 앱이 Privy의 `signMessage` API를 사용하여 증명 메시지(`lasersell-register:<pubkey>:<timestamp>`)에 서명합니다. 이 ed25519 서명은 LaserSell 서버에 의해 검증되어 사용자가 해당 지갑을 제어하고 있음을 증명합니다. 이는 데스크톱 앱에서 사용되는 것과 동일한 검증 흐름입니다.

## 트랜잭션 흐름

자동 매도가 트리거될 때:

1. LaserSell의 모니터링 서버가 전략 임계값 도달을 감지
2. 미서명 트랜잭션이 생성됨 (데스크톱과 동일)
3. LaserSell이 Privy에 트랜잭션 **서명**을 요청 (서명만, 제출 아님)
4. Privy가 TEE 내부에서 서명하고 서명된 트랜잭션을 반환
5. LaserSell이 자체 RPC를 통해 서명된 트랜잭션을 Solana에 제출
6. 사용자에게 트랜잭션 서명이 포함된 알림 전송

어떤 시점에서도 LaserSell 서버는 개인키를 볼 수 없습니다. 서명과 키 저장은 전적으로 Privy의 인프라 내에서 이루어집니다.

## Privy에서 키 제거

Privy 인프라에서 개인키를 완전히 제거하려면:

### 가져온 지갑
1. 대시보드의 **Wallet** 페이지로 이동
2. 가져온 지갑 카드의 휴지통 아이콘을 클릭
3. 이렇게 하면 LaserSell에서 지갑이 제거되고 Privy의 TEE에서 키가 삭제됩니다
4. 키는 보안 영역에서 영구적으로 파기됩니다

### 임베디드 지갑
임베디드 지갑은 Privy 계정에 연결되어 있습니다. 제거하려면:
1. 먼저 **개인키를 내보내세요** (지갑 카드에서 "Export Key" 클릭)
2. 내보낸 키를 개인 지갑(Phantom, Backpack 등)에 안전하게 보관하세요
3. [고객 지원](https://docs.lasersell.io/getting-support/support)에 연락하여 Privy 계정 삭제를 요청하세요
4. Privy 계정이 삭제되면 임베디드 지갑 키는 영구적으로 파기됩니다

<Warning>
  지갑 또는 Privy 계정 삭제는 되돌릴 수 없습니다. 개인키를 내보내지 않으면 해당 지갑의 자금에 대한 접근을 영구적으로 잃게 됩니다.
</Warning>
