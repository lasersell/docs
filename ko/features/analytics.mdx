LaserSell은 프로토콜을 통해 수행한 모든 거래를 추적하고 실행 가능한 성과 데이터로 변환합니다. 분석 대시보드는 [Desktop 앱](/desktop/analytics)과 API를 통해 이용할 수 있습니다.

> **참고:** 모든 분석, 인사이트 및 업적은 LaserSell을 통한 거래 활동만을 기반으로 합니다. 다른 도구를 통하거나 온체인에서 직접 실행한 거래는 포함되지 않습니다.

## 기간

모든 지표는 네 가지 기간으로 조회할 수 있습니다:

- **24시간** — 오늘의 성과
- **7일** — 주간 스냅샷
- **30일** — 월간 개요
- **전체 기간** — 전체 거래 히스토리

각 기간은 독립적인 지표 세트를 계산하므로, 시간에 따른 거래 변화를 비교할 수 있습니다.

## 주요 지표

### 총 손익

선택한 기간의 모든 청산된 포지션에 대한 순 실현 수익 또는 손실입니다. 각 거래의 최종 수익 합계이며, 선택한 표시 통화로 표시됩니다.

### 총 거래량

해당 기간의 모든 진입 금액(토큰 매수에 사용한 SOL 또는 USD1)의 합계입니다. 결과와 관계없이 배치한 자본의 양을 측정합니다.

### 승률

수익으로 마감한 거래의 비율입니다.

```
승률 = 수익 거래 / (수익 거래 + 손실 거래) × 100
```

청산된 포지션만 계산됩니다. 열린 포지션은 승률 계산에서 제외됩니다.

### 총 거래 수

선택한 기간 동안 청산된 포지션의 수입니다.

### 평균 손익

청산된 거래당 평균 수익 또는 손실입니다. 일반적인 거래가 수익성이 있는지, 얼마나 수익성이 있는지 알려줍니다.

### 평균 보유 시간

청산 전략이든 수동이든 포지션을 청산하기까지 평균적으로 보유하는 시간입니다. 시간과 분으로 표시됩니다.

짧은 평균 보유 시간은 보통 빠르게 움직이는 밈코인 거래를 나타냅니다. 긴 보유 시간은 더 큰 시가총액의 토큰을 거래하거나 더 넓은 트레일링 스탑을 사용하고 있음을 나타낼 수 있습니다.

### 최고 거래

해당 기간에서 가장 높은 실현 수익을 기록한 단일 거래입니다. 상승 잠재력을 이해하고 전략이 수익을 충분히 달리게 하는지 파악하는 데 유용합니다.

### 최악 거래

해당 기간에서 가장 깊은 실현 손실을 기록한 단일 거래입니다. 이 수치가 평균 손익에 비해 지나치게 크다면 손절매를 더 촘촘히 설정해야 할 수 있습니다.

## 일일 손익 차트

90일간의 일일 순 수익 또는 손실을 보여주는 막대 차트입니다. 녹색 막대는 수익 일, 빨간 막대는 손실 일입니다. 막대 위에 마우스를 올리면 해당 일의 정확한 수치와 거래 수를 볼 수 있습니다.

이 차트는 패턴을 파악하는 데 도움이 됩니다: 지속적으로 수익을 내고 있는지, 몇 번의 큰 날이 평균을 견인하는지? 손실 일이 시간이 지남에 따라 줄어들고 있는지?

## 청산 사유 분석

선택한 기간 동안 청산을 트리거한 이유의 시각적 분석입니다:

| 사유 | 의미 |
| :--- | :--- |
| **목표 수익** | 목표 수익 임계값에 도달함 |
| **트레일링 스탑** | 가격이 최고점에서 트레일링 비율만큼 하락함 |
| **손절매** | 가격이 손절매 임계값에 도달함 |
| **졸업** | 토큰이 본딩 커브에서 DEX로 마이그레이션됨 |
| **데드라인** | 설정된 데드라인 후 포지션이 타임아웃됨 |

이 분석은 전략을 조정하는 데 가장 유용한 도구 중 하나입니다. 예를 들어:

- 대부분의 청산이 **손절매**라면, 손절이 너무 촘촘하거나 나쁜 시점에 진입하고 있을 수 있습니다.
- 대부분의 청산이 **데드라인**이라면, 포지션이 횡보하고 있을 수 있습니다. 거래하는 토큰이 전략에 충분한 변동성을 가지고 있는지 고려해 보세요.
- **트레일링 스탑**이 지배적이고 손익이 건전하다면, 전략이 잘 작동하고 있는 것입니다. 상승을 포착하고 하락에서 청산합니다.
- **목표 수익**이 자주 발동하지만 최고 거래가 목표보다 훨씬 높다면, 수익 기회를 놓치고 있을 수 있습니다. 목표를 넓히거나 트레일링 스탑으로 전환하는 것을 고려해 보세요.

## API 접근

분석은 LaserSell API를 통해 프로그래밍 방식으로 이용할 수 있습니다:

```
GET /v1/stats
```

위에 나열된 모든 지표와 함께 네 가지 기간을 모두 반환합니다. 응답 형식에 대한 자세한 내용은 [API 문서](/api/exit-api/history)를 참조하세요.
