Solana 토큰과 상호작용하는 AI 에이전트는 LaserSell을 실행 및 청산 관리 레이어로 사용할 수 있습니다. 스왑 라우팅, 프로토콜 처리, 청산 로직을 처음부터 구축하는 대신, 에이전트가 LaserSell에 연결하면 사전 구축된 트랜잭션, 실시간 손익 추적, 자동 청산을 즉시 이용할 수 있습니다.

## 에이전트가 LaserSell으로 할 수 있는 것

**거래 빌드 및 실행.** [REST API](/api/overview)는 지원되는 모든 Solana DEX 또는 런치패드에 대한 서명되지 않은 스왑 트랜잭션을 반환합니다. 에이전트가 민트 주소와 금액을 보내면 트랜잭션을 받아 서명하고 제출합니다. 프로토콜별 로직이 필요 없습니다.

**전략으로 청산 자동화.** [Exit Intelligence Stream](/api/stream/overview)은 지갑을 실시간으로 감시하고 설정 가능한 임계값(목표 수익, 손절매, 트레일링 스탑, 졸업 시 매도, 데드라인 타임아웃)이 충족되면 사전 구축된 청산 트랜잭션을 전달합니다. 에이전트가 한 번 연결하면 자동으로 청산 신호를 받습니다.

**포지션 모니터링.** 스트림은 지속적인 손익 업데이트, 잔액 변경, 포지션 오픈 및 클로즈를 제공합니다. 에이전트는 이 데이터를 의사 결정, 사용자에게 정보 표시, 다른 시스템에 피드하는 데 사용할 수 있습니다.

**모든 주요 Solana 프로토콜 지원.** 단일 통합으로 Pump.fun, PumpSwap, Meteora DBC, Meteora DAMM V2, Raydium Launchpad, Raydium CPMM을 지원합니다. 에이전트는 토큰이 어떤 프로토콜에 있는지 알 필요가 없습니다.

## 연결 방법

AI 에이전트가 LaserSell과 함께 작동하는 세 가지 방법이 있습니다:

### 1. MCP 서버 (문서 접근)

LaserSell MCP 서버는 MCP 호환 AI 클라이언트(Claude, Cursor, Windsurf 등)에게 실시간으로 LaserSell 문서를 검색하고 검색할 수 있는 기능을 제공합니다. 작업 중에 API 스키마, SDK 패턴, 코드 예제 또는 통합 세부사항을 조회해야 하는 에이전트에 유용합니다.

설정 지침은 [MCP 서버](/ai-agents/mcp-server)를 참조하세요.

### 2. 코딩 스킬 (내장 SDK 지식)

LaserSell 코딩 스킬을 설치하면 AI 어시스턴트에게 LaserSell 코드를 작성할 때 컨텍스트에 직접 로드되는 SDK 패턴, 메서드 시그니처, 예제, 문제 해결에 대한 깊은 지식을 제공합니다.

설치 지침은 [코딩 스킬](/ai-agents/coding-skills)을 참조하세요.

### 3. SDK 및 API (직접 통합)

트랜잭션을 빌드하거나, 포지션을 관리하거나, 프로그래밍 방식으로 거래를 실행해야 하는 에이전트의 경우, LaserSell SDK 또는 API를 직접 통합하세요.

- 전체 API 표면에 대한 [API 개요](/api/overview).
- 첫 번째 트랜잭션을 빌드하기 위한 [빠른 시작](/api/quickstart).
- [TypeScript](/api/sdk/typescript), [Python](/api/sdk/python), [Rust](/api/sdk/rust), [Go](/api/sdk/go)로 SDK 이용 가능.

## 에이전트가 LaserSell을 사용해야 하는 이유

### 에이전트 통합 문제

오늘날 Solana 토큰을 거래하려는 AI 에이전트는 일련의 어려운 문제를 해결해야 합니다: 토큰이 어떤 DEX 프로토콜에 있는지 식별하고, 온체인 풀 상태를 역직렬화하고, 프로토콜별 수학을 사용하여 스왑 금액을 계산하고, 올바른 인스트럭션 세트를 구성하고, 우선 수수료를 관리하고, 슬리피지 재시도를 처리하고, 청산할 시기를 알기 위해 가격 변화를 폴링해야 합니다. 각 프로토콜(Pump.fun, PumpSwap, Raydium, Meteora)은 다른 프로그램, 다른 계정 레이아웃, 다른 스왑 로직을 가지고 있습니다. 이것은 수백 시간의 통합 작업이며, 새로운 런치패드가 출시될 때마다 깨집니다.

LaserSell은 이 모든 것을 두 개의 API 호출로 축소합니다: 트랜잭션 빌드를 위한 하나, 청산 신호를 받기 위한 WebSocket 하나.

### 기술적 이점

| 기능 | LaserSell 없이 | LaserSell과 함께 |
|---|---|---|
| **프로토콜 라우팅** | 에이전트가 토큰이 어떤 DEX에 있는지 감지하고, 풀 상태를 역직렬화하고, 6개 이상의 프로그램 각각에 대해 프로토콜별 인스트럭션을 빌드해야 함 | 단일 API 호출. 민트 주소를 보내면 서명 준비가 된 트랜잭션을 받음. LaserSell이 서버 측에서 프로토콜, 풀, 경로를 해결 |
| **청산 로직** | 에이전트가 가격 데이터를 위해 RPC를 폴링하고, 손익을 계산하고, 트레일링 스탑을 위해 최고 수익을 추적하고, 데드라인을 위한 타이머를 관리하고, 수동으로 매도 트랜잭션을 빌드해야 함 | Exit Intelligence Stream에 한 번 연결. 전략 조건이 발동될 때 사전 구축된 서명되지 않은 트랜잭션이 포함된 `exit_signal_with_tx` 이벤트를 수신 |
| **손익 추적** | 에이전트가 토큰 잔액을 가져오고, 소수점을 해석하고, 풀 준비금을 쿼리하고, 현물 가격을 계산하고, 정기적인 간격으로 손익을 유도해야 함 | 스트림이 지속적으로 `pnl_update` 이벤트를 전달. 폴링 없음, RPC 호출 없음, 가격 계산 없음 |
| **새 포지션 감지** | 에이전트가 지갑 트랜잭션 히스토리를 모니터링하거나 온체인 로그를 구독하고 각 트랜잭션을 파싱하여 매수를 식별해야 함 | 구매가 랜딩하는 순간 전체 컨텍스트(진입 비용, 민트, 마켓 컨텍스트)와 함께 `position_opened`를 스트림이 발동 |
| **멀티 프로토콜 지원** | Pump.fun 본딩 커브, PumpSwap AMM, Raydium Launchpad, Raydium CPMM, Meteora DBC, Meteora DAMM V2에 대한 별도 통합 | 하나의 통합으로 6개 모두 지원. 새 프로토콜이 추가되어도 에이전트 코드가 변경되지 않음 |
| **트랜잭션 서명** | 해당 없음 | 모든 트랜잭션이 서명되지 않은 상태로 반환됨. 에이전트가 로컬에서 서명. 개인키가 에이전트 환경을 떠나지 않음 |
| **슬리피지 처리** | 에이전트가 슬리피지를 추정하고, 실패 시 재시도하고, 허용 범위를 점진적으로 높여야 함 | 서버가 슬리피지를 계산. SDK에서 설정 가능한 범프 로직으로 클라이언트 재시도가 자동 |
| **새 런치패드 지원** | 에이전트가 각 새 프로그램을 리버스 엔지니어링하고, 처음부터 인스트럭션을 빌드하고, 지속적인 호환성을 유지해야 함 | LaserSell이 서버 측에서 프로토콜 지원을 추가. 에이전트는 코드 변경 없이 지원을 받음 |

### 실제로 의미하는 것

**RPC 호출 감소.** LaserSell 없이 에이전트는 가격 데이터를 폴링하고, 잔액을 가져오고, 풀 상태를 지속적으로 읽어야 합니다. Exit Intelligence Stream을 사용하면 서버가 모든 것을 푸시합니다. 에이전트의 RPC 사용은 트랜잭션 제출로만 줄어듭니다.

**결정론적 청산 실행.** 전략 조건(TP, SL, 트레일링 스탑, 데드라인, 졸업)은 모든 가격 틱에서 서버 측에서 평가됩니다. 에이전트가 지속적으로 온라인 상태를 유지하거나 가격 급등을 놓칠 걱정을 할 필요가 없습니다. 임계값이 충족되면 청산 트랜잭션이 이미 빌드되어 대기 중입니다.

**프로토콜별 코드 없음.** 에이전트는 `{ mint, user_pubkey, amount }`를 `/v1/sell` 또는 `/v1/buy`로 보냅니다. 토큰이 Pump.fun 본딩 커브, PumpSwap AMM, Raydium CPMM 풀 중 어디에 있든 반환된 트랜잭션이 작동합니다.

**이벤트 기반 아키텍처.** 스트림은 에이전트에 자연스럽게 적합합니다. 폴링 루프를 실행하는 대신 에이전트가 구독하고 반응합니다: 새 거래를 로깅하기 위한 `position_opened`, 데이터를 표면화하기 위한 `pnl_update`, 서명하고 제출하기 위한 `exit_signal_with_tx`. 낭비되는 연산 없는 깔끔한 비동기 패턴입니다.

**4개 언어에서 SDK 패리티.** TypeScript, Python, Rust, Go SDK는 동일한 스키마, 메서드 이름, 동작을 노출합니다. Python으로 구축된 에이전트는 Rust로 구축된 에이전트와 동일하게 작동합니다. 어떤 SDK도 이등 시민이 아닙니다.
